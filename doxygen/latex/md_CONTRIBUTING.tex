\hypertarget{md_CONTRIBUTING_autotoc_md13}{}\doxysection{How to contribute?}\label{md_CONTRIBUTING_autotoc_md13}
Contributing to the project is very easy and every little bit counts! Just follow the following steps\+:

To contribute to this project, follow the next steps\+:


\begin{DoxyItemize}
\item {\itshape Fork} (just to external contributors)
\item Create \href{CONTRIBUTING.md\#issues}{\texttt{ {\itshape issues}}}
\item Create \href{CONTRIBUTING.md\#branches-policy}{\texttt{ {\itshape branchs}}}
\item Follow the commit policy \href{CONTRIBUTING.md\#commit-policy}{\texttt{ {\itshape commits}}}
\item Submit \href{CONTRIBUTING.md\#merges-and-pull-requests-policy}{\texttt{ {\itshape Pull Request}}}
\end{DoxyItemize}\hypertarget{md_CONTRIBUTING_autotoc_md14}{}\doxysubsection{Commit Policy}\label{md_CONTRIBUTING_autotoc_md14}
The {\itshape issues} need title, description, at least one asigned, {\itshape labels}, {\itshape milestoner} (the {\itshape sprint} to conclude)

The Labels used in the project are described in the topic \href{https://github.com/PDA-FGA/Playground/labels}{\texttt{ Labels}}.

To create a issue follow task template or user history template\hypertarget{md_CONTRIBUTING_autotoc_md15}{}\doxysubsection{Branches Policy}\label{md_CONTRIBUTING_autotoc_md15}
\hypertarget{md_CONTRIBUTING_autotoc_md16}{}\doxysubsubsection{$<$em$>$main$<$/em$>$}\label{md_CONTRIBUTING_autotoc_md16}
The {\itshape main} is the production branch, where the stable version of the project will be. It will be closed for commits and for pushes. See the merge policy in the topic \href{CONTRIBUTING.md\#merges-to-main}{\texttt{ Merges para {\itshape master}}}.\hypertarget{md_CONTRIBUTING_autotoc_md17}{}\doxysubsubsection{Branch Names}\label{md_CONTRIBUTING_autotoc_md17}
The development branches will be created from the {\itshape main} branch with the default naming {\ttfamily x/issue\+\_\+name}, where {\ttfamily x} represents the issue tracking code.\hypertarget{md_CONTRIBUTING_autotoc_md18}{}\doxysubsection{Commits Policy}\label{md_CONTRIBUTING_autotoc_md18}
All commits must be mention the issue, for that, just add {\ttfamily \#\texorpdfstring{$<$}{<}issue\+\_\+number\texorpdfstring{$>$}{>}}.
